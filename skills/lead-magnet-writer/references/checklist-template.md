# Шаблон чек-листа-лидмагнита

## Параметры
- Объём: 300–500 слов + пункты (читается за 5–8 минут — правило 15 минут)
- Пунктов: 7–15 (оптимально 10–12)
- Тон: конкретный, практичный — как инструкция от профи
- **Ключевой принцип**: бьёт в пассивную боль, даёт алгоритм, открывает новую проблему перед CTA

---

## Когда выбирать этот формат

Чек-лист работает, когда:
- Аудитория на уровне 3 (Solution Aware) — знает о проблеме, ищет систему действий
- Тема имеет чёткую последовательность шагов
- Читатель занят, хочет быстрый результат
- Нужно что-то, что легко сохранить и переслать

---

## Структура (6 блоков)

---

### 1. Заголовок (1 строка)

**Обязательно**: число в заголовке.

Формулы:
```
[Число] шагов / пунктов / критериев, чтобы [результат]
→ «10 шагов для подготовки консультации, которая закрывается в продажу»

Чек-лист: как [действие] без [нежелательного]
→ «Чек-лист: как написать оффер без "дорого" и "подумаю"»

Проверь себя: [число] вопросов / признаков / критериев
→ «Проверь свой оффер: 8 вопросов, которые покажут, почему клиенты уходят думать»
```

**Тест на пассивную боль**: заголовок должен вызывать мысль «о, это про меня» — не «мне нужна помощь прямо сейчас».

---

### 2. Список выгод (3–4 строки, перед пунктами)

Сразу после заголовка. Создаёт ощущение ценности.

Шаблон:
```
Этот чек-лист поможет тебе:
— [конкретная выгода 1]
— [конкретная выгода 2]
— [конкретная выгода 3]
```

---

### 3. Вступление (2–4 предложения)

Зеркало пассивной боли — кратко.

Шаблон:
```
Этот чек-лист для [кого именно].

Если ты [ситуация, которую читатель узнаёт] — он поможет тебе [результат].

[Одна фраза о том, почему это работает — конкретно, без воды].
```

---

### 4. Блоки чек-листа

Разбей пункты на 2–4 смысловых раздела. Каждый раздел — отдельная часть работы.

**Формат раздела:**
```
## [Название раздела] — конкретная задача, не тема

☐ [Пункт — глагол + конкретное действие или проверяемый критерий]
☐ [Пункт]
☐ [Пункт]
```

**Принципы пунктов:**
- Каждый пункт — одно действие или утверждение, которое можно поставить галочку
- Начинай с глагола: «Определи», «Напиши», «Проверь», «Убедись, что...»
- Если нужно пояснение — добавь 1 строку курсивом под пунктом
- Не перегружай объяснениями — пункт должен читаться за 3 секунды

**Хорошо:**
```
☐ Напиши оффер в одном предложении: «Я помогаю [кому] [результат] за [срок/деньги]»
```

**Плохо:**
```
☐ Работа с оффером (слишком абстрактно)
```

**Социальное доказательство** — добавь одну строку в конце одного из разделов:
```
*Этот шаг пропускают 7 из 10 экспертов — и теряют продажи на финальном этапе.*
```

---

### 5. БЛОК: Открытие новой проблемы ⚡

**Это обязательный блок.** Между последним пунктом чек-листа и CTA.

Логика: читатель применит чек-лист и достигнет промежуточного результата — но увидит следующую задачу, с которой уже нужна помощь эксперта.

Шаблон:
```
Когда ты пройдёшь эти [N] пунктов — ты [промежуточный результат].

Но дальше встанет вопрос: [новая проблема].
Именно здесь большинство останавливаются — потому что [причина].

[1–2 предложения: почему это требует другого уровня / экспертной помощи].
```

Пример:
```
Когда ты пройдёшь эти 10 шагов — у тебя будет готовый, чёткий оффер.

Но дальше встанет вопрос: как его доносить на живой консультации, когда клиент начинает сомневаться.
Именно здесь большинство теряют продажи — не из-за цены, а из-за того, как они разговаривают.

Это уже другой навык. И именно с ним я работаю с клиентами.
```

---

### 6. CTA (40–60 слов)

Привязан к новой проблеме, открытой выше.

Шаблон:
```
Если хочешь разобраться с [новая проблема] — напиши мне [кодовое слово / способ связи].

Мы созвонимся на [время] и я покажу,
как [конкретный результат] работает именно в твоей ситуации.

[Снятие страха: бесплатно / без давления / без обязательств].
```

---

## Сравнение: Статья vs Чек-лист

| Критерий | Статья | Чек-лист |
|----------|--------|----------|
| Длина | 800–1400 слов | 300–500 слов + пункты |
| Структура | Нарративная, с историями | Пошаговая, нумерованная |
| Тон | Глубокий, аргументированный | Практичный, быстро к делу |
| Читают | Чтобы понять «почему» | Чтобы знать «что делать» |
| Лучше для | Уровни 1–2 осознанности | Уровень 3 осознанности |
| Виральность | Ниже | Выше — легко переслать |

---

## Принципы живого чек-листа

1. **Конкретность**: «Напиши оффер в одном предложении» — не «поработай с оффером»
2. **Проверяемость**: на каждый пункт есть ответ «выполнено / нет»
3. **Прогрессия**: пункты идут в логическом порядке — от первого к последнему
4. **Не перегружай**: до 15 пунктов. Больше — уже не читают полностью
5. **Один раздел = одна задача**: группируй пункты по смыслу

---

## Финальная проверка перед отправкой

- [ ] Заголовок содержит число
- [ ] Есть список выгод перед пунктами
- [ ] Есть хотя бы одно социальное доказательство (одна строка)
- [ ] Есть блок «открытие новой проблемы» перед CTA
- [ ] CTA привязан к новой проблеме
- [ ] Общий объём ≤15 пунктов, читается ≤10 минут
