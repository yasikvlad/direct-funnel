# Шаблоны Telegram-сообщений

## ⚠️ ЖЁСТКОЕ ОГРАНИЧЕНИЕ ЦЕПОЧКИ

Цепочка = ровно 4 сообщения в Telegram. Ни больше.

1. Выдача лидмагнита
2. Дожим до лидмагнита (через 20–30 минут)
3. Продающее сообщение 1 — раскрытие продукта
4. Продающее сообщение 2 — финальный CTA

Никакого прогрева на 3 дня. Никаких post-event серий. Только эти 4.

---

## Принципы всей цепочки

- Тон: живой, человечный, не корпоративный
- Длина: 4–8 предложений на сообщение
- Продажа — через конкретные результаты и живую демонстрацию
- Эмодзи: 1–2 на сообщение, органично
- Читай и опирайся на примеры, шаблоны - это не строгие материалы, а просто структура для того, чтобы оттолкнуться.

## Сообщение 2 — Дожим до лидмагнита

*Время отправки: через 20–30 минут после выдачи*

Задача: побудить человека реально открыть материал прямо сейчас, не отложить.
Прочитай примеры, чтобы увидеть как нужно делать реальное внедрение.

### Структура
1. Переход: небольшое наблюдение или провокационный тезис
2. **Два сценария провала** в нише — узнаваемые, конкретные:
   - Цикл, из которого не выйти
   - Ловушка откладывания / типичная ошибка
3. Лидмагнит — это выход за пределы обоих сценариев
4. Финал: «Именно это внутри»

Длина: 5–7 предложений. Тон: вдумчивый, без давления.

### Шаблон
```
[Переход / наблюдение]

[Сценарий 1: типичный цикл без выхода в нише]

[Сценарий 2: ловушка, которая выглядит разумно, но не работает]

[Название лидмагнита] — конкретный выход: [как именно].

Именно это внутри.
```

```
Работать руками там, где можно использовать ии-агента. Сейчас поясню 👇🏻
```

## Сообщение 3 — Продающее сообщение 1 (раскрытие продукта)

*Время отправки: на следующий день после выдачи лидмагнита*

Задача: мягко раскрыть основной продукт через логику «лидмагнит дал тебе А, продукт даёт А+Б+В».

### Структура
1. Открытие через результат или живое наблюдение — что получают после лидмагнита
2. Переход: лидмагнит — часть системы, дальше — следующий уровень
3. Конкретное описание продукта — что именно получит человек, с числами
4. Один кейс клиента — имя/образ, ситуация до, результат после (числа)
5. Мягкое упоминание продукта без жёсткого CTA

### Шаблон
```
[Открытие через типичную реакцию после материала или живой результат]

[Переход к продукту — логика «это был первый шаг»]

Только представь как может выглядеть [ситуация после]:
— [результат 1]
— [результат 2]
— [результат 3]

[Кейс: имя, было/стало с цифрами]

[Мягкое упоминание продукта — что это и что внутри, без давления]
```

## Сообщение 4 — Финальный CTA (заявка / покупка)

*Время отправки: через 1–2 дня после сообщения 3*

Задача: дать конкретный, чёткий призыв к действию. Никакого давления, только ясность.

### Структура
1. Прямое обращение или вопрос-наблюдение
2. Что конкретно внутри продукта — 3 конкретных результата (буллеты)
3. Дата / формат / цена — прозрачно
4. Чёткий CTA: одно действие
5. Снятие возражения (без обязательств / просто посмотри)

### Шаблон
```
[Обращение — прямой вопрос или наблюдение]

[Что конкретно получит человек]:
— [результат 1]
— [результат 2]
— [результат 3]

[Дата / формат / цена]

[CTA: одно конкретное действие — написать слово / нажать кнопку]

[Снятие страха — 1 строка]
```

### 🥇 Золотой стандарт (реальный пример)
```

Дата — 18 марта.
```

## Кодовые слова — рекомендации

Выбирай кодовое слово так, чтобы оно:
- Было связано с темой лидмагнита или услуги
- Легко запоминалось
- Не требовало объяснений

Примеры по нишам:
- AI/автоматизация: «АГЕНТ», «МИДЛ», «СИСТЕМА», «ИИ»
- Маркетинг/воронки: «ОФФЕР», «КЛИЕНТЫ», «ПЛАН», «ВОРОНКА»
- Коуч/психолог: «ХОЧУ», «СТАРТ», «ДА»
- Образование: «УЧИТЬСЯ», «КУРС», «ГАЙД»
